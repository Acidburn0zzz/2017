Pulse 0.1.1 Release Notes

17th November, 2014 — George MacRorie

Just a quick update to tell you all about a release that is about to take place (0.1.1). Pulse will automatically upgrade itself soon, if not already. Unless otherwise specified within the settings panel (cog icon in the top right).

This release is in response to an issue with communication between Pulse instances and Syncthing instances. The TLS configuration signs its host as pulse on a Pulse device and syncthing on a Syncthing device. To retain the ability to communicate with Syncthing devices, release 0.1.1 will set hosts in certs to syncthing by default on Pulse set up. As well as allowing certs with a host pulse that have been created, to still be supported by Pulse 0.1.0+ clients.

For users of Pulse 0.1.0, that want to connect with a Syncthing device, regeneration of your configuration certificates will be required until we come to a shared solution with the Syncthing community. In order to do this we recommend you upgrade to 0.1.1, delete the cert.pem and key.pem from your Pulse configuration folder. Then restart. Be aware this will generate you a new Device ID. Default configuration file locations are as follows. Please try ~/.config if you have no success with $XDG_CONFIG_HOME on Linux.

  • Mac OSX: /Library/Application Support/Pulse/
  • Windows: %localappdata%\Pulse\
  • Linux: $XDG_CONFIG_HOME/pulse/
  • Linux: ~/.config/pulse/
